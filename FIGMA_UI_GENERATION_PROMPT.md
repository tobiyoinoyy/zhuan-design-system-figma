# Figma UI Generation Prompt

用于新建对话时，让 AI 基于转转设计系统仓库和 Figma libraries，把低保真图生成高保真 UI 设计稿。

目标仓库：

<https://github.com/tobiyoinoyy/zhuan-design-system-figma>

## 可直接复制的 Prompt

```text
请先读取 GitHub 仓库 tobiyoinoyy/zhuan-design-system-figma 里的转转设计系统代码、组件目录、组件使用规则和示例，再基于我提供的低保真图生成高保真 UI 设计稿。

核心要求：

1. 不要直接按低保真图手动画页面。必须先理解仓库里的组件体系、页面结构、设计 token、图标、业务组件和已有示例。
2. 先分析当前页面里每一个 UI 元素应该映射到哪些 GitHub 代码组件或 Figma libraries 组件。
3. 如果需要先给我预览，请先用仓库里的组件逻辑渲染 HTML 高保真预览；我确认 HTML 后，再写入 Figma。
4. 写入 Figma 时必须优先搜索并调用 Figma libraries 里的真实组件实例。凡是 GitHub 组件库或 Figma libraries 里已有、相似或可映射的元素，都必须优先调用组件。
5. 组件调用检查必须覆盖全页面每一个 UI 元素，而不是只检查几个重点区域。包括但不限于状态栏、导航栏、标题栏、搜索、Tab、排序筛选、标签、卡片、列表项、商品图、价格、营销标签、图标、按钮、弹层、底部操作区、Toast、提示条、空状态、加载态等。
6. 如果没有完全匹配的组件，可以基于相似库组件创建派生组件，但派生组件内部也必须尽量复用已有库组件、图标、文本样式和变量，不要用矩形、文字、手画图标随意拼出来。
7. 页面主要容器必须使用合理的 auto layout，尤其是导航、Tab、筛选区、卡片列表、弹层、滚动内容、底部按钮区和安全区。
8. 图层命名要能看出组件来源和映射关系，例如：
   - ZHUAN Design / 排序面板=收起 ← git: SortTabs
   - ZHUAN Design / 大按钮=单排 ← git: ButtonBar(single)
   - ZZ Derived / ChannelSearchNav ← based on ZHUAN 搜索栏 + 返回 + 搜索
9. 写入完成后必须验证：
   - 应该是组件的图层是否真的是 INSTANCE。
   - 每个主要组件是否来自 Figma libraries 或派生组件。
   - 派生组件内部是否复用了库组件。
   - auto layout 是否正确，而不是纯坐标硬摆。
   - 是否存在明显低保真残留、错位、遮挡、按钮位置异常、文字溢出或组件未调用的问题。
10. 最后请给我一份简短说明，列出：
   - 本页面调用了哪些 Figma library 组件。
   - 哪些是基于库组件创建的派生组件。
   - 哪些元素没有找到可用库组件，只能临时自建，并说明原因。
```

## 推荐工作流程

1. 读取仓库结构  
   先查看组件目录、文档、示例和 token，不要直接开始画。

2. 建立组件映射表  
   对低保真图逐层拆解，列出每个区域对应的 GitHub 组件和 Figma library 组件。

3. 生成 HTML 预览  
   如果需要预览，先用仓库组件逻辑还原高保真 HTML，让用户确认视觉方向。

4. 写入 Figma  
   写入时不要只照着 HTML 截图重画。应使用 Figma libraries 里的组件实例，缺失时再创建派生组件。

5. 验证结构  
   写入后检查实例数量、组件来源、派生组件内部结构、auto layout 层级和最终截图。

## 组件调用原则

- 能调用真实组件实例，就不要手画。
- 能用 library 图标，就不要自己画 SVG 或用文本符号替代。
- 能用已有搜索栏、按钮、Tab、筛选、卡片、弹层、状态栏，就不要用矩形拼。
- 没有完全匹配时，可以创建派生组件，但派生组件内部仍然要尽量调用已有组件。
- 页面图层应能让设计师一眼看出组件来源，而不是只有视觉相似。

## Figma 验收标准

- 页面不是单纯“画出来的”，而是尽量由真实组件实例组成。
- 每一个应该能调用组件的地方都已经调用组件。
- 派生组件内部也复用了已有库组件。
- auto layout 层级清楚，主要区域可维护。
- 底部按钮区、弹层、滚动内容、安全区位置自然，不靠硬摆。
- 搜索、筛选、排序、图标、按钮、卡片等高频组件在图层中能看到明确 INSTANCE。
- 最终视觉达到高保真，不保留低保真占位感。

## 常见错误

- 只根据低保真图重画矩形和文字，没有调用组件。
- HTML 预览用了组件逻辑，但写入 Figma 时丢失组件映射。
- 只检查状态栏、搜索栏、底部按钮等显眼组件，忽略列表项、标签、图标、Toast、卡片内部元素。
- 外观像组件，但图层不是 INSTANCE。
- 做了派生组件，但派生组件内部仍然是手画元素。
- 底部按钮区没有按 Sheet/footer/safe area 结构做 auto layout。
- 用普通文字或手画线条替代图标库。
