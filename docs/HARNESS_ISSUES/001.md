## 一、原始反馈

产物视觉上明显偏离转转工具型页面的中性风格:

- 删除分组、特征标签、隐私分类的左侧图标统一被刷成 `var(--zd-color-link)` 蓝色
- 整页看起来像低保真稿里"彩色分类图标 + 大色块"的搬运,而不是转转工具型页面应有的"中性 + 仅在重点处出彩"

低保真输入源里图标本来就是彩色装饰画,agent 把这个视觉规则直接照搬,token 用对了变量(`--zd-color-link` 在白名单里),但用错了语义位——`--zd-color-link` 是"链接 / 信息入口色",不是"分类装饰色"。

> 注:该案例发生在旧仓库(token 名为 `--zz-link`),迁移至本仓库时统一改写为新 token 名 `--zd-color-link`。规则结论不变。

## 三、维护者裁判

### 是不是真问题

是。不是审美偏好,也不是单次特例——只要"低保真稿带彩色分类图标 + 工具型页面"这个组合再出现一次,agent 就会重复同样的错。

### 该不该进 Harness

进。来源等级 `confirmed`(维护者亲自实测,产物路径、对话 id、复现条件都齐)。

### 类型归属

B 类。Harness 现有的"用没用 token / 用没用真组件"是结构层校验,不能覆盖语义层(token 用在对的场景)。语义边界靠 case 列举永远列不完,这次先把入口卡住:

- 工具型页面默认图标走中性色 `var(--zd-color-text-tertiary)`
- token 语义白名单(`--zd-color-link` / `--zd-color-primary` / `--zd-color-text/-secondary/-tertiary/-muted`)首批进 INPUT_SOURCE_RULES,后续每次踩坑追加,不一次铺死
- 自查清单加一条兜底:批量出现 `--zd-color-link` / `--zd-color-primary` 着色 Icon(≥2 处)需要 agent 自检并在 DECISIONS.md 说明理由,否则视为规则违反

## 五、对伙伴的最终回复

> 这次工具型页面的图标被刷成蓝色,根因是低保真稿里的彩色分类图标被 agent 当成"工具页该有的视觉风格"照搬。
>
> 已经在规则里加了两条:
> 1. 工具型页面默认图标走中性灰,不再吃低保真上的彩色装饰
> 2. agent 用 `--zd-color-link` / `--zd-color-primary` 着色图标超过 2 处,需要在 DECISIONS.md 说明理由,否则自查不通过
>
> 已生效,你下次重新生成同类页面应该不会再出现整片蓝色图标。如果还出,在反馈里说"HI-001 没修干净",我会加证据复核。
